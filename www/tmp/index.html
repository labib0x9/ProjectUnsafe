// What We're Building

A Hands-On Security
Learning Environment

ProjectUnsafe is a controlled, sandboxed environment designed for students, researchers, and administrators to study real-world vulnerabilities in a safe, ethical space. We believe understanding how attacks work is the first step to building robust defenses.

From web exploits to low-level memory bugs, each module walks you through the mechanics, the impact, and the mitigation — turning theory into hands-on practice without putting real systems at risk.

// Core Topics

What You Will Learn

Web Security

XSS → 4 labs

Cross-Site Scripting (XSS) lets attackers inject malicious scripts into web pages viewed by other users. These scripts can steal cookies, hijack sessions, or redirect users — all without touching the server. We cover reflected, stored, and DOM-based XSS with live demos.

Database Attack

SQL Injection → 4 labs

SQL Injection manipulates database queries by inserting malicious SQL code through user inputs. Attackers can bypass authentication, dump entire databases, or even execute OS commands. Learn how parameterized queries and ORMs prevent this class of attack.

Memory Error

Segmentation Fault → 4 labs

A segfault occurs when a program accesses memory it has no right to — a common sign of buffer overflows or dangling pointers. In security, these bugs are doorways to arbitrary code execution. We explore stack smashing, heap corruption, and safe coding practices in C/C++.

OS & Shell

Linux → 4 labs

Linux is the backbone of servers, embedded systems, and security tools worldwide. Understanding file permissions, process management, shell scripting, and privilege escalation is essential for both offense and defense. Get comfortable in the terminal — it's your primary weapon.
